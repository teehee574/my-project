----------
Короткий гайд по Git для твого проекту
1. Перевірити статус файлів
Показує, які файли змінені, додані або готові до коміту.

bash
Копировать
Редактировать
git status
2. Додати зміни у відстеження (до коміту)
Додає всі змінені файли:

bash
Копировать
Редактировать
git add .
Або конкретний файл:

bash
Копировать
Редактировать
git add ім'я_файлу
3. Зробити коміт
Зафіксувати додані зміни з коментарем:

bash
Копировать
Редактировать
git commit -m "Короткий опис змін"
4. Відправити зміни на GitHub
bash
Копировать
Редактировать
git push
(перший раз, якщо не встановлено відстеження, додай -u origin main)

5. Завантажити оновлення з GitHub
Якщо хтось змінив проект, або ти працюєш на іншому комп’ютері:

bash
Копировать
Редактировать
git pull
6. Переглянути всі гілки
bash
Копировать
Редактировать
git branch
7. Перейти на іншу гілку
bash
Копировать
Редактировать
git checkout ім'я_гілки
8. Створити і перейти на нову гілку
bash
Копировать
Редактировать
git checkout -b ім'я_нової_гілки
9. Перейменувати поточну гілку (наприклад master → main)
bash
Копировать
Редактировать
git branch -M main
10. Подивитись історію комітів
bash
Копировать
Редактировать
git log
Рекомендація по роботі:
Перед початком роботи оновлюй локальний репозиторій командою git pull.

Після внесення змін додавай їх (git add), коміть (git commit) і пуш (git push).

Якщо хочеш працювати над новою функцією — створи гілку (git checkout -b feature-branch), а потім змерджи її в main.
